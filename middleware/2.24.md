什么是分布式计算



#### 远程过程调用 RPC

1. cs模型  使用消息传递来实现  marshalling  unmarshalling   打包  解包
2. 常用属性  请求和响应同步  分布式分布的透明性  多种可靠性的保证
3. 缺点  紧耦合    缺少服务的概念

#### 面向对象的中间件体系结构 OOM

1. 可以是本地或远程  **远程方法调用**   **对象的参照**可以是本地或远程  远程对象必须有一个可见的**远程接口**  通过**代理对象** 远程对象可以像本地对象一样使用
2. 优点  定位的透明性  
3. 缺点  请求和响应同步作用  分布式废弃内存回收？  静态&重量级

#### 面向消息中间件  MOM

1. 消息保存在消息队列中   消息格式
2. 属性  非同步 松耦合 消息是队列化的  提供**可靠**的服务  消息处理由中间的消息服务器来完成   （对数据库的集成比较自然）
3. 缺点   对于编程的抽象比较贫乏   消息的格式对中间件是未知的   队列的抽象值给予一对一的通讯
4. Web Services  XML
